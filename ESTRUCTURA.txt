app.py
------
Aplicacion Flask con TODAS las rutas HTTP (sin SQL crudo):

Autenticacion:
  GET  /            -> Redirige a dashboard
  GET  /login       -> Formulario login
  POST /login       -> Valida credenciales
  GET  /logout      -> Cierra sesion
  GET  /dashboard   -> Panel principal

Generacion de fichas:
  POST /api/generar           -> Inicia scraping asincronico
  GET  /api/stream/<job_id>   -> Stream de progreso en tiempo real (SSE)
  GET  /propiedad/<id>        -> Visualiza ficha generada
  GET  /p/<token>             -> Acceso publico por token (sirve la pagina directamente, no redirect)

Propiedades:
  GET    /propiedades                          -> Lista propiedades
  PUT    /api/propiedades/<id>/tags             -> Actualiza tags
  DELETE /api/propiedades/<id>                  -> Soft-delete (papelera)
  POST   /api/propiedades/<id>/restaurar        -> Restaura de papelera
  DELETE /api/propiedades/<id>/eliminar-definitivo -> Elimina permanentemente
  GET    /api/propiedades/papelera              -> Lista papelera
  DELETE /api/propiedades/papelera/vaciar       -> Vacia papelera
  DELETE /api/propiedades                       -> Borra todas las activas

Clientes:
  GET    /api/clientes                    -> Lista (paginado + busqueda + filtro estado)
  POST   /api/clientes                    -> Crea
  PUT    /api/clientes/<id>               -> Edita
  DELETE /api/clientes/<id>               -> Soft-delete
  POST   /api/clientes/<id>/restaurar     -> Restaura
  DELETE /api/clientes/<id>/eliminar-definitivo -> Elimina permanentemente
  GET    /api/clientes/<id>/actividad     -> Lista actividad
  POST   /api/clientes/<id>/actividad     -> Agrega actividad
  GET    /api/clientes/papelera           -> Lista papelera
  DELETE /api/clientes/papelera/vaciar    -> Vacia papelera

Intereses:
  POST   /api/intereses                   -> Marca propiedad de interes
  DELETE /api/intereses                   -> Quita interes
  GET    /api/intereses/cliente/<id>      -> Intereses de un cliente
  GET    /api/intereses/propiedad/<id>    -> Interesados en propiedad

Administracion:
  POST /api/perfil              -> Actualiza perfil
  POST /api/cambiar_password    -> Cambia contrasena
  GET  /api/admin/usuarios      -> Lista usuarios (admin)
  POST /api/admin/crear_usuario -> Crea usuario (admin)
  POST /api/admin/toggle_usuario -> Activa/desactiva usuario
  POST /api/admin/reset_password -> Reset contrasena (admin)
  POST /api/admin/delete_usuario -> Elimina usuario (admin)

Utilidades:
  GET /proxy-image              -> Proxy de imagenes (evita CORS/hotlinking)


config.py
---------
Configuracion centralizada:
  SECRET_KEY, FIRECRAWL_API_KEY, DEBUG, rutas estaticas


db.py
-----
Gestion SQLite con optimizaciones:
  - get_connection()   -> Reutiliza conexion por request via Flask g
  - init_db()          -> Crea tablas
  - _migrate_*()       -> Migraciones con early-exit

Tablas:
  1. users                      -> Usuario + contrasena hash
  2. properties                 -> Propiedades scrapeadas + token publico + tags
  3. clients                    -> Clientes con pipeline de estados
  4. client_property_interests  -> Relaciones cliente-propiedad

---------------------------
Scraping automatico de ZonaProp, Argenprop, MercadoLibre, REMAX.

================================================================================
                        FLUJO PRINCIPAL
================================================================================

GENERACION DE FICHA:
  1. Usuario: Pega URL(s) en dashboard (hasta 5)
  2. JavaScript: POST /api/generar
  3. Flask: Crea job asincronico
  4. Thread paralelo:
     scraper_service.scrape(url)
       -> Extrae todos los datos
     property_service.create_property()
       +-- Guarda en BD (con token publico)
       +-- Descarga fotos a static/properties/{id}/
       +-- Renombra (01.jpg, 02.jpg, etc.)
  5. JavaScript: GET /api/stream/{job_id} (SSE)
       -> Recibe logs en tiempo real
  6. Ficha disponible en:
       /propiedad/{id} (interna)
       /p/{token} (publica, con Open Graph)


PIPELINE DE CLIENTES:
  nuevo_lead -> contactado -> visito_propiedad -> cerrado
                                               -> perdido

  Acciones: llamar, enviar_propiedades, coordinar_visita,
            seguimiento, esperar_respuesta

================================================================================
                        COMO EMPEZAR
================================================================================

1. Instalar Python 3.10+

2. Instalar dependencias:
   pip install -r requirements.txt

3. Instalar Playwright (para scraping):
   python -m playwright install chromium
   python -m playwright install-deps chromium

4. Configurar .env:
   echo "SECRET_KEY=$(openssl rand -hex 32)" > .env

5. Iniciar app:
   python app.py

6. Abrir navegador:
   http://localhost:8080
   usuario: admin
   contrasena: admin123
